ONDEX/OVTK scripting reference

Author: Artem Lysenko, artem.lysenko_at_rothamsted.ac.uk

Ondex scripting engine allows flexible manipulation of graphs and annotation at run-time and is implemented in JavaScript. More information about Javascript syntax and data types can be found on the following sites:

http://en.wikipedia.org/wiki/JavaScript

http://en.wikipedia.org/wiki/JavaScript_syntax

http://www.crockford.com/javascript/javascript.html

A useful guide about the differences between JavaScript and Java:

http://peter.michaux.ca/article/5004

It is backed by the Mozilla Rhino scripting engine that allows native Java classes to be used alongside JavaSctript objects. For more information about Rhino-specific features and how to use them please refer to their website: http://www.mozilla.org/rhino/.

All of the necessary functions and objects needed to access data elements, graph and attributes of concepts and relations can be accessed via scripting interface. In OVTK all commands work on the graph associated with the currently selected viewer.

At the moment all of the ONDEX data types are wrapped in a standardised way all of the getters with no argument and setter that take one argument become properties in JavaScript representation and can be used with assignment operator '=' to write and read from them. All other methods become functions and are used with a standard Java notation e.g. fn(arg0, arg1, arg2). There are also convenience methods added that allow terms of controlled vocabulary, like accessions, attribute names, etc. to be supplied as a string as well as an original data type. For example to get concept Attribute: concept.getConceptGDS("TAXID"). Internally the strings are resolved against the metadata as per norm. When testing new Java code in OVTK, it can be made into a new global function by adding it as a static method to net.sourceforge.ondex.ovtk2.ui.scripting.OVTK2CustomFunctions class.

Here is the list of all currently supported options and how they can be called from the scripting interface. For the full documentation of what particular methods do please refer to the ONDEX/OVTK JavaDoc and look for the original methods/classes that back the JavaScript representation. If you add new features and need to regenerate the documentation, delete 'Scripting_ref.htm' (this file) to have it regenerated with the new information included.
